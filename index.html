What the runtime does

The core product is not an OpenCode plugin. It is a host-agnostic memory runtime with a thin OpenCode demo surface for real validation.

Recall

Explainable retrieval

Retrieves project facts, task memory, decisions, and summaries using scoped filters, keyword scoring, tags, kinds, and SQLite FTS5 BM25 when available.

Governance

Memory lifecycle controls

Consolidates exact duplicates, reinforces recurring facts, supersedes stale decisions, reports stats, and prepares dry-run pruning plans before deletion exists.

Evidence

Auditable artifacts

Writes prompt documents, recall logs, recall metrics, portable snapshots, prune plans, and validation results for final report evidence.

Runtime-first architecture

The design keeps durable memory separate from host integration so agent-specific tooling can change without moving the source of truth.

  1. 1Store
    Persist memory items locally in SQLite.
  2. 2Retrieve
    Score and rank context with transparent reasons.
  3. 3Govern
    Reinforce, supersede, snapshot, and plan pruning.
  4. 4Inject
    Build bounded OpenCode-readable prompt documents.
RAG component

Lightweight retrieval-augmented generation

The knowledge base is local memory. Retrieval uses explicit project/task scope and explainable ranking instead of requiring a hosted vector database. OpenCode receives a bounded context bundle, while `recall-log.json` keeps the audit trail out of the prompt.
